Preguntas frecuentes

¿Necesito Tor o I2P para usar esta app?

No. Clearnet funciona por sí solo y te da acceso a más de 50.000 emisoras a través de la API de RadioBrowser, como una app de radio normal. Tor e I2P son completamente opcionales para usuarios que desean privacidad o acceso a emisoras de la darknet.

¿Cómo mantengo mi privacidad/anonimato al usar esta app?

Por defecto, la app se conecta directamente desde tu dirección IP. Para permanecer anónimo, necesitas activar un modo forzado en los ajustes y/o ejecutar una VPN en segundo plano. Hay varias opciones según tu configuración:

  • Forzar Tor para todo — Enruta todo el tráfico (transmisiones, llamadas API, carátulas) a través de Tor. Nada toca Internet directamente. Si Tor se desconecta, todo el tráfico se bloquea — no hay respaldos a Clearnet.
  • Forzar Tor excepto I2P — Enruta el tráfico de Clearnet a través de Tor, pero envía el tráfico I2P a través del proxy HTTP de I2P. Esto evita la latencia innecesaria de enrutar I2P sobre Tor.
  • Forzar proxy personalizado — Enruta todo el tráfico a través de un proxy que tú configuras (SOCKS4, SOCKS5, HTTP o HTTPS). Útil si ejecutas tu propio proxy o usas un proveedor de proxy que no es Tor.
  • Forzar proxy personalizado excepto Tor/I2P — Enruta el tráfico de Clearnet a través de tu proxy personalizado, mientras que las emisoras de Tor e I2P usan sus proxies nativos.

Para máxima privacidad: activa un modo forzado, desactiva la API de RadioBrowser si no necesitas emisoras de Clearnet, desactiva la carga de carátulas para eliminar todas las solicitudes de imágenes, activa el cifrado de la base de datos para proteger tus datos locales en reposo, y ejecuta una VPN en segundo plano junto con el modo proxy de InviZible Pro como capa extra de protección si no confías plenamente en la app o en alguna de las APIs a las que se conecta.

Todos los modos forzados son a prueba de fallos — si el proxy se cae, el tráfico se bloquea, no se filtra. Esto incluye la prevención de fugas de DNS — en los modos forzados, todas las consultas DNS se resuelven a través del proxy en lugar del DNS de tu sistema/ISP.

Si no confías en esta app, ejecuta una VPN de confianza en segundo plano.

¿Qué datos recopila esta app?

Ninguno. Todos los datos (contraseñas de proxy, emisoras, historial, ajustes, favoritos) se almacenan localmente en tu dispositivo. No hay sincronización en la nube, ni analíticas, ni telemetría. Opcionalmente puedes cifrar todos los datos locales con SQLCipher.

¿Las emisoras de Clearnet bloquean Tor en el modo forzado de Tor?

A veces, pero raramente. La gran mayoría de las emisoras de Clearnet no bloquean Tor.

¿Las carátulas se cargan a través de mi proxy?

Sí. Las imágenes de las emisoras se cargan a través del mismo canal de proxy que las transmisiones en las tres redes (Clearnet, I2P y Tor). En los modos forzados, todas las solicitudes de imágenes incluyen protección completa contra fugas de DNS — el DNS se resuelve a través del proxy, no del resolvedor del sistema. Si el proxy se desconecta, la carga de imágenes se bloquea en lugar de recurrir a una conexión directa. También puedes desactivar las carátulas por completo en los ajustes para eliminar todas las solicitudes de imágenes.

¿Activar Tor (modo no forzado) oculta mi IP?

No necesariamente. Sin activar un modo forzado, todo el tráfico de Clearnet (transmisiones, carátulas) se cargará desde tu IP directa. Activar este modo no forzado solo te permite acceder a emisoras de Tor fácilmente sin el esfuerzo adicional de enrutar todo a través de Tor.

¿Cómo importo emisoras de otra app?

Ajustes → Importar. Formatos compatibles: CSV, JSON, M3U, PLS.

¿Por qué está habilitado el tráfico en texto claro en el manifiesto?

Los sitios I2P (.i2p) y Tor (.onion) usan http:// en lugar de https://. El tráfico en texto claro debe estar habilitado para que estos dominios funcionen.

¿Puedo usar un proxy diferente en lugar de InviZible Pro?

Sí. Cualquier proxy SOCKS5 que enrute a través de Tor funciona. También puedes configurar proxies SOCKS4, SOCKS5, HTTP o HTTPS con autenticación opcional en los ajustes de la app.

¿Por qué SOCKS4 no protege completamente mi privacidad como SOCKS5 y otros?

SOCKS4 no soporta resolución DNS remota. Al usar el modo forzar proxy personalizado con un proxy SOCKS4, las consultas DNS se resuelven localmente a través del DNS de tu sistema/ISP, lo que puede filtrar los dominios a los que te conectas. SOCKS5 resuelve el DNS a través del propio proxy, previniendo estas fugas. Si la privacidad DNS te importa, usa SOCKS5, HTTP o HTTPS en lugar de SOCKS4 para tu proxy personalizado. La app mostrará una advertencia cuando selecciones SOCKS4 como recordatorio.

¿Cómo envío emisoras de la darknet?

Envía emisoras .i2p y .onion a través de la API del Registro de Radio. Las emisoras de Clearnet van a la base de datos de RadioBrowser.

Instalación

Descarga la última APK aquí o usa Obtainium para actualizaciones automáticas.

Verificar

Las versiones están firmadas con Sigstore. Para verificar, descarga tanto el archivo .apk como el .apk.sigstore de la misma versión, luego ejecuta desde el directorio que contiene los archivos descargados:

cd ~/Downloads
cosign verify-blob \
  --bundle deutsia-radio-v1.6.5.apk.sigstore \
  --certificate-identity siewargottlichschon@gmail.com \
  --certificate-oidc-issuer https://github.com/login/oauth \
  deutsia-radio-v1.6.5.apk

Compilar

git clone https://github.com/deutsia/deutsia-radio.git
cd deutsia-radio
./gradlew assembleDebug

Requiere Java 21 LTS. Establece JAVA_HOME=/usr/lib/jvm/java-21-openjdk antes de compilar.
