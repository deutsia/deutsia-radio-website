FAQ

Ai-je besoin de Tor ou I2P pour utiliser cette application ?

Non. Le Clearnet fonctionne seul et vous donne accès à plus de 50 000 stations via l'API RadioBrowser, comme une application radio classique. Tor et I2P sont entièrement optionnels pour les utilisateurs souhaitant plus de confidentialité ou accéder aux stations du darknet.

Comment rester privé/anonyme en utilisant cette application ?

Par défaut, l'application se connecte directement depuis votre adresse IP. Pour rester anonyme, vous devez activer un mode forcé dans les paramètres et/ou exécuter un VPN en arrière-plan. Plusieurs options s'offrent à vous selon votre configuration :

  • Forcer Tor pour tout — Achemine tout le trafic (flux, appels API, pochettes d'albums) via Tor. Rien ne touche Internet directement. Si Tor se déconnecte, tout le trafic est bloqué — il n'y a aucun repli sur le clearnet.
  • Forcer Tor sauf I2P — Achemine le trafic clearnet via Tor, mais envoie le trafic I2P via le proxy HTTP I2P. Cela évite la latence inutile du routage I2P via Tor.
  • Forcer le proxy personnalisé — Achemine tout le trafic via un proxy que vous configurez (SOCKS4, SOCKS5, HTTP ou HTTPS). Utile si vous utilisez votre propre proxy ou un fournisseur de proxy non-Tor.
  • Forcer le proxy personnalisé sauf Tor/I2P — Achemine le trafic clearnet via votre proxy personnalisé, tandis que les stations Tor et I2P utilisent leurs proxys natifs.

Pour une confidentialité maximale : activez un mode forcé, désactivez l'API RadioBrowser si vous n'avez pas besoin des stations clearnet, désactivez le chargement des pochettes pour éliminer toutes les requêtes d'images, activez le chiffrement de la base de données pour protéger vos données locales au repos, et exécutez un VPN en arrière-plan en parallèle du mode proxy d'InviZible Pro pour une couche de protection supplémentaire si vous ne faites pas entièrement confiance à l'application ou aux API auxquelles elle se connecte.

Tous les modes forcés sont à sécurité intégrée — si le proxy tombe en panne, le trafic est bloqué, pas divulgué. Cela inclut la prévention des fuites DNS — en modes forcés, toutes les requêtes DNS sont résolues via le proxy plutôt que par le DNS de votre système/FAI.

Si vous ne faites pas confiance à cette application, exécutez un VPN de confiance en arrière-plan.

Quelles données cette application collecte-t-elle ?

Aucune. Toutes les données (mots de passe proxy, stations, historique, paramètres, favoris) sont stockées localement sur votre appareil. Il n'y a pas de synchronisation cloud, pas d'analyse et pas de télémétrie. Vous pouvez optionnellement chiffrer toutes les données locales avec SQLCipher.

Les stations clearnet bloquent-elles Tor en mode Tor forcé ?

Parfois, mais rarement. La grande majorité des stations clearnet ne bloquent pas Tor.

Les pochettes d'albums se chargent-elles via mon proxy ?

Oui. Les pochettes des stations sont chargées via le même pipeline proxy que les flux sur les trois réseaux (clearnet, I2P et Tor). En modes forcés, toutes les requêtes d'images incluent une protection complète contre les fuites DNS — le DNS est résolu via le proxy, pas par votre résolveur système. Si le proxy se déconnecte, le chargement des images est bloqué plutôt que de basculer sur une connexion directe. Vous pouvez également désactiver entièrement les pochettes dans les paramètres pour éliminer toutes les requêtes d'images.

L'activation de Tor (mode non forcé) masque-t-elle mon IP ?

Pas nécessairement. Sans activer un mode forcé, tout le trafic clearnet (flux, pochettes) sera chargé depuis votre IP directe. Activer ce mode non forcé vous permet simplement d'accéder facilement aux stations Tor sans avoir à acheminer tout le trafic via Tor.

Comment importer des stations depuis une autre application ?

Paramètres → Importer. Formats pris en charge : CSV, JSON, M3U, PLS.

Pourquoi le trafic en clair est-il activé dans le manifeste ?

Les sites I2P (.i2p) et Tor (.onion) utilisent http:// plutôt que https://. Le trafic en clair doit être activé pour que ces domaines fonctionnent.

Puis-je utiliser un autre proxy qu'InviZible Pro ?

Oui. Tout proxy SOCKS5 routant via Tor fonctionne. Vous pouvez également configurer des proxys SOCKS4, SOCKS5, HTTP ou HTTPS avec authentification optionnelle dans les paramètres de l'application.

Pourquoi SOCKS4 ne protège-t-il pas entièrement ma vie privée comme SOCKS5 et les autres ?

SOCKS4 ne prend pas en charge la résolution DNS distante. Lors de l'utilisation de Forcer le proxy personnalisé avec un proxy SOCKS4, les requêtes DNS sont résolues localement via le DNS de votre système/FAI, ce qui peut divulguer les domaines auxquels vous vous connectez. SOCKS5 résout le DNS via le proxy lui-même, empêchant ces fuites. Si la confidentialité DNS vous importe, utilisez SOCKS5, HTTP ou HTTPS au lieu de SOCKS4 pour votre proxy personnalisé. L'application affichera un avertissement lorsque vous sélectionnez SOCKS4 en guise de rappel.

Comment soumettre des stations darknet ?

Soumettez des stations .i2p et .onion via l'API Radio Registry. Les stations clearnet vont dans la base de données RadioBrowser.

Installer

Téléchargez le dernier APK ici ou utilisez Obtainium pour les mises à jour automatiques.

Vérifier

Les versions sont signées avec Sigstore. Pour vérifier, téléchargez les fichiers .apk et .apk.sigstore de la même version, puis exécutez depuis le répertoire contenant les fichiers téléchargés :

cd ~/Downloads
cosign verify-blob \
  --bundle deutsia-radio-v1.6.5.apk.sigstore \
  --certificate-identity siewargottlichschon@gmail.com \
  --certificate-oidc-issuer https://github.com/login/oauth \
  deutsia-radio-v1.6.5.apk

Compiler

git clone https://github.com/deutsia/deutsia-radio.git
cd deutsia-radio
./gradlew assembleDebug

Nécessite Java 21 LTS. Définissez JAVA_HOME=/usr/lib/jvm/java-21-openjdk avant de compiler.
