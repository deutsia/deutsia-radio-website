FAQ

ဤအက်ပ်ကိုသုံးရန် Tor သို့မဟုတ် I2P လိုအပ်ပါသလား?

မလိုပါ။ Clearnet သည် သီးသန့်အလုပ်လုပ်ပြီး သာမန်ရေဒီယိုအက်ပ်ကဲ့သို့ RadioBrowser API မှတစ်ဆင့် ရေဒီယိုဘူတာ 50,000 ကျော်ကို ဝင်ရောက်နိုင်သည်။ Tor နှင့် I2P သည် ကိုယ်ရေးလုံခြုံမှုလိုချင်သော သို့မဟုတ် darknet ရေဒီယိုဘူတာများဝင်ရောက်လိုသော အသုံးပြုသူများအတွက် ရွေးချယ်စရာသာဖြစ်သည်။

ဤအက်ပ်ကိုအသုံးပြုစဉ် ကိုယ်ရေးလုံခြုံမှု/အမည်ဝှက်ကို ဘယ်လိုထိန်းသိမ်းရမလဲ?

မူရင်းအတိုင်းအားဖြင့်၊ အက်ပ်သည် သင့် IP လိပ်စာမှ တိုက်ရိုက်ချိတ်ဆက်သည်။ အမည်ဝှက်ဖြစ်ရန်၊ ဆက်တင်များတွင် အတင်းအကျပ်မုဒ်ကိုဖွင့်ရန် နှင့်/သို့မဟုတ် နောက်ခံတွင် VPN ကိုအသုံးပြုရန် လိုအပ်သည်။ သင့်စနစ်ပေါ်မူတည်၍ ရွေးချယ်စရာများစွာရှိသည်:

  • Tor အားလုံးအတင်းအကျပ် — အသွားအလာအားလုံး (စတရိမ်များ၊ API ခေါ်ဆိုမှုများ၊ မျက်နှာဖုံးအနုပညာ) ကို Tor မှတစ်ဆင့် ပို့ဆောင်သည်။ အင်တာနက်ကို တိုက်ရိုက်ထိတွေ့သည်မရှိပါ။ Tor ချိတ်ဆက်မှုပြတ်ပါက အသွားအလာအားလုံးကို ပိတ်ဆို့သည် — clearnet ပြန်ကျမှုမရှိပါ။
  • I2P မှလွဲ၍ Tor အတင်းအကျပ် — Clearnet အသွားအလာကို Tor မှတစ်ဆင့် ပို့ဆောင်သော်လည်း I2P အသွားအလာကို I2P HTTP ပရောက်စီမှတစ်ဆင့် ပို့ဆောင်သည်။ I2P ကို Tor မှတစ်ဆင့် ပို့ဆောင်ခြင်း၏ မလိုအပ်သော နှောင့်နှေးမှုကို ရှောင်ရှားသည်။
  • စိတ်ကြိုက်ပရောက်စီ အတင်းအကျပ် — အသွားအလာအားလုံးကို သင်ပြင်ဆင်ထားသော ပရောက်စီ (SOCKS4၊ SOCKS5၊ HTTP သို့မဟုတ် HTTPS) မှတစ်ဆင့် ပို့ဆောင်သည်။ ကိုယ်ပိုင်ပရောက်စီအသုံးပြုသူ သို့မဟုတ် Tor မဟုတ်သော ပရောက်စီဝန်ဆောင်မှုသုံးသူများအတွက် အသုံးဝင်သည်။
  • Tor/I2P မှလွဲ၍ စိတ်ကြိုက်ပရောက်စီ အတင်းအကျပ် — Clearnet အသွားအလာကို သင့်စိတ်ကြိုက်ပရောက်စီမှတစ်ဆင့် ပို့ဆောင်ပြီး Tor နှင့် I2P ရေဒီယိုဘူတာများသည် ၎င်းတို့၏ မူလပရောက်စီများကို အသုံးပြုသည်။

အမြင့်ဆုံးကိုယ်ရေးလုံခြုံမှုအတွက်: အတင်းအကျပ်မုဒ်ကိုဖွင့်ပါ၊ clearnet ရေဒီယိုဘူတာများမလိုအပ်ပါက RadioBrowser API ကိုပိတ်ပါ၊ ပုံတောင်းဆိုမှုအားလုံးကိုဖယ်ရှားရန် မျက်နှာဖုံးအနုပညာတင်ခြင်းကိုပိတ်ပါ၊ လိုကယ်ဒေတာကိုကာကွယ်ရန် ဒေတာဘေ့စ်ကုဒ်ဝှက်ခြင်းကိုဖွင့်ပါ၊ အက်ပ် သို့မဟုတ် ချိတ်ဆက်သော API များကို အပြည့်အဝမယုံကြည်ပါက InviZible Pro ၏ ပရောက်စီမုဒ်နှင့်အတူ နောက်ခံတွင် VPN ကိုအသုံးပြုပါ။

အတင်းအကျပ်မုဒ်အားလုံးသည် ချို့ယွင်းမှုကင်းလွတ်သည် — ပရောက်စီကျသွားပါက အသွားအလာကို ယိုစိမ့်စေခြင်းမပြုဘဲ ပိတ်ဆို့သည်။ ၎င်းတွင် DNS ယိုစိမ့်မှုကာကွယ်ခြင်းပါဝင်သည် — အတင်းအကျပ်မုဒ်များတွင် DNS စုံစမ်းမှုအားလုံးကို သင့်စနစ်/ISP DNS မှမဟုတ်ဘဲ ပရောက်စီမှတစ်ဆင့် ဖြေရှင်းသည်။

ဤအက်ပ်ကို မယုံကြည်ပါက၊ ယုံကြည်ရသော VPN ကို နောက်ခံတွင်အသုံးပြုပါ။

ဤအက်ပ်သည် မည်သည့်ဒေတာကို စုဆောင်းသနည်း?

ဘာမှမစုဆောင်းပါ။ ဒေတာအားလုံး (ပရောက်စီစကားဝှက်များ၊ ရေဒီယိုဘူတာများ၊ မှတ်တမ်း၊ ဆက်တင်များ၊ စိတ်ကြိုက်များ) ကို သင့်စက်တွင် လိုကယ်သိုလှောင်ထားသည်။ cloud sync မရှိ၊ analytics မရှိ၊ telemetry မရှိပါ။ SQLCipher ဖြင့် လိုကယ်ဒေတာအားလုံးကို ရွေးချယ်ကုဒ်ဝှက်နိုင်သည်။

Tor အတင်းအကျပ်မုဒ်တွင် clearnet ရေဒီယိုဘူတာများက Tor ကို ပိတ်ဆို့ပါသလား?

တစ်ခါတစ်ရံ ဖြစ်သော်လည်း ရှားပါးသည်။ Clearnet ရေဒီယိုဘူတာအများစုသည် Tor ကို ပိတ်ဆို့ခြင်းမရှိပါ။

မျက်နှာဖုံးအနုပညာသည် ပရောက်စီမှတစ်ဆင့် တင်ပါသလား?

ဟုတ်ကဲ့။ ရေဒီယိုဘူတာအနုပညာကို ကွန်ရက်သုံးခုစလုံး (clearnet၊ I2P နှင့် Tor) တွင် စတရိမ်များနှင့်အတူတူ ပရောက်စီပိုက်လိုင်းမှတစ်ဆင့် တင်သည်။ အတင်းအကျပ်မုဒ်များတွင် ပုံတောင်းဆိုမှုအားလုံးတွင် DNS ယိုစိမ့်မှု အပြည့်အဝကာကွယ်ခြင်းပါဝင်သည် — DNS ကို သင့်စနစ်ဖြေရှင်းသူမှမဟုတ်ဘဲ ပရောက်စီမှတစ်ဆင့် ဖြေရှင်းသည်။ ပရောက်စီချိတ်ဆက်မှုပြတ်ပါက တိုက်ရိုက်ချိတ်ဆက်မှုသို့ ပြန်ကျခြင်းမပြုဘဲ ပုံတင်ခြင်းကို ပိတ်ဆို့သည်။ ပုံတောင်းဆိုမှုအားလုံးကို ဖယ်ရှားရန် ဆက်တင်များတွင် မျက်နှာဖုံးအနုပညာကို လုံးဝပိတ်နိုင်သည်။

Tor ဖွင့်ခြင်း (အတင်းအကျပ်မဟုတ်သောမုဒ်) သည် ကျွန်ုပ်၏ IP ကို ဝှက်ပါသလား?

မူလအားဖြင့် မဝှက်ပါ။ အတင်းအကျပ်မုဒ်ကိုမဖွင့်ဘဲ clearnet အသွားအလာအားလုံး (စတရိမ်များ၊ မျက်နှာဖုံးအနုပညာ) ကို သင့်တိုက်ရိုက် IP မှ တင်မည်ဖြစ်သည်။ ဤအတင်းအကျပ်မဟုတ်သောမုဒ်ကိုဖွင့်ခြင်းသည် အရာအားလုံးကို Tor မှတစ်ဆင့် ပို့ဆောင်ရန် အပိုအားထုတ်ခြင်းမရှိဘဲ Tor ရေဒီယိုဘူတာများကို လွယ်ကူစွာဝင်ရောက်နိုင်ရန်သာ ဖြစ်သည်။

အခြားအက်ပ်မှ ရေဒီယိုဘူတာများကို မည်သို့ထည့်သွင်းရမည်နည်း?

ဆက်တင်များ → ထည့်သွင်းရန်။ ပံ့ပိုးသောဖော်မတ်များ: CSV, JSON, M3U, PLS။

manifest တွင် cleartext အသွားအလာကို ဘာကြောင့်ဖွင့်ထားသနည်း?

I2P (.i2p) နှင့် Tor (.onion) ဆိုဒ်များသည် https:// အစား http:// ကိုအသုံးပြုသည်။ ဤဒိုမိန်းများအလုပ်လုပ်ရန် cleartext အသွားအလာကို ဖွင့်ထားရမည်။

InviZible Pro အစား အခြားပရောက်စီကို သုံးနိုင်ပါသလား?

ဟုတ်ကဲ့။ Tor မှတစ်ဆင့် ပို့ဆောင်သော SOCKS5 ပရောက်စီမည်သည်မဆို အလုပ်လုပ်သည်။ အက်ပ်ဆက်တင်များတွင် ရွေးချယ်နိုင်သော auth ဖြင့် SOCKS4၊ SOCKS5၊ HTTP သို့မဟုတ် HTTPS ပရောက်စီများကိုလည်း ပြင်ဆင်နိုင်သည်။

SOCKS4 သည် SOCKS5 နှင့် အခြားများကဲ့သို့ ကိုယ်ရေးလုံခြုံမှုကို ဘာကြောင့် အပြည့်အဝမကာကွယ်နိုင်သနည်း?

SOCKS4 သည် အဝေးထိန်း DNS ဖြေရှင်းခြင်းကို ပံ့ပိုးမထားပါ။ SOCKS4 ပရောက်စီဖြင့် စိတ်ကြိုက်ပရောက်စီ အတင်းအကျပ်မုဒ်ကို အသုံးပြုသောအခါ DNS စုံစမ်းမှုများကို သင့်စနစ်/ISP DNS မှတစ်ဆင့် လိုကယ်ဖြေရှင်းပြီး သင်ချိတ်ဆက်နေသော ဒိုမိန်းများ ယိုစိမ့်နိုင်သည်။ SOCKS5 သည် ပရောက်စီကိုယ်တိုင်မှ DNS ကို ဖြေရှင်းပြီး ဤယိုစိမ့်မှုများကို ကာကွယ်သည်။ DNS ကိုယ်ရေးလုံခြုံမှု အရေးကြီးပါက သင့်စိတ်ကြိုက်ပရောက်စီအတွက် SOCKS4 အစား SOCKS5၊ HTTP သို့မဟုတ် HTTPS ကိုအသုံးပြုပါ။ SOCKS4 ကိုရွေးချယ်သောအခါ အက်ပ်သည် သတိပေးချက်ပြမည်။

Darknet ရေဒီယိုဘူတာများကို မည်သို့တင်သွင်းရမည်နည်း?

Radio Registry API မှတစ်ဆင့် .i2p နှင့် .onion ရေဒီယိုဘူတာများကို တင်သွင်းပါ။ Clearnet ရေဒီယိုဘူတာများကို RadioBrowser ဒေတာဘေ့စ် သို့ တင်သွင်းပါ။

ထည့်သွင်းခြင်း

နောက်ဆုံး APK ကို ဤနေရာ မှ ဒေါင်းလုဒ်လုပ်ပါ သို့မဟုတ် အလိုအလျောက်အပ်ဒိတ်အတွက် Obtainium ကိုအသုံးပြုပါ။

အတည်ပြုခြင်း

ထုတ်ဝေမှုများကို Sigstore ဖြင့် လက်မှတ်ရေးထိုးထားသည်။ အတည်ပြုရန်၊ တူညီသော ထုတ်ဝေမှုမှ .apk နှင့် .apk.sigstore ဖိုင်နှစ်ခုလုံးကို ဒေါင်းလုဒ်လုပ်ပြီး ဒေါင်းလုဒ်လုပ်ထားသောဖိုင်များပါဝင်သော ဖိုင်တွဲမှ အောက်ပါကိုအသုံးပြုပါ:

cd ~/Downloads
cosign verify-blob \
  --bundle deutsia-radio-v1.6.5.apk.sigstore \
  --certificate-identity siewargottlichschon@gmail.com \
  --certificate-oidc-issuer https://github.com/login/oauth \
  deutsia-radio-v1.6.5.apk

တည်ဆောက်ခြင်း

git clone https://github.com/deutsia/deutsia-radio.git
cd deutsia-radio
./gradlew assembleDebug
